硬链接
ln old new
硬链接没有"源文件"和"链接名"之分，两者完全平等，A 和 B 指向同一个 inode（二者创建时间、修改时间都一样），删除A，B仍可用；删除B，A仍可用，没有任何元数据记录谁是"原始文件"，rm一个文件名，是对应inode减1，减到0才真正删除一个文件。 
不能跨磁盘创建，
编译一个文件，相当于也在编辑另一个文件，
不能对目录创建，
ls -lai 显示的是文件，能看到每个文件名所对应的inode号及该inode有多少个硬链接
cp 一个硬链接 之后得到的新文件就是普通文件了


软链接/符号链接
ln -s old new   #old最好写绝对路径 -sf是强制覆盖旧链接
ln -lai显示的是链接，符号链接本身是一个新的inode
