进阶梯队（Python核心机制与并发）
- Python 中 async/await 和 yield 有什么区别？为什么在处理大模型流式输出时常用 yield，而在调用多个外部 API 时推荐用 async/await？

考察点：生成器 vs 协程、I/O 密集型任务处理。

评估标准：能区分 yield 用于生成数据序列（惰性求值），async/await 用于处理异步I/O。理解流式输出是“数据生成”场景，而并发API调用是“I/O等待”场景。

- Python 的全局解释器锁（GIL）对多线程有什么影响？在开发需要并发处理多个 LLM 请求的 Agent 服务时，你会如何设计架构来规避 GIL 的限制？

考察点：GIL 的理解、多进程/协程的应用。

评估标准：能说出 GIL 导致多线程无法充分利用多核CPU。解决方案是采用 多进程（multiprocessing）或 异步I/O（asyncio）来处理并发的网络请求（LLM API调用属于I/O密集型）。

- Python 的垃圾回收（GC）主要机制是什么？如果一个长期运行的 Agent 服务出现了内存缓慢增长，你可能从哪些方面排查？

考察点：引用计数、循环引用、内存问题排查。

评估标准：知道引用计数和循环引用检测。排查方向包括：全局变量或缓存未及时清理、循环引用、大对象（如对话历史列表）未分片或摘要。
